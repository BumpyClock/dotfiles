# Fluent Design and WinUI Guidance

Use for visual polish or Windows design alignment.

## Design hub essentials
- Start from Windows design hub: layout, navigation, input, typography, motion.
- Keep consistency across devices, input types, form factors.

## Windows 11 design principles
- Effortless: reduce common-task steps; make primary actions easy to find.
- Calm: limit noise; emphasize only what matters.
- Personal: respect theme, contrast, text size; allow customization.
- Familiar: prefer standard WinUI controls/patterns unless strong reason.
- Complete + coherent: consistent spacing, typography, motion.

## Signature experiences checklist
- Color: show hierarchy; draw attention only when needed.
- Elevation/layering: show hierarchy + surface relationships.
- Iconography: clear familiar symbols for recognition + scanning.
- Materials: Mica/Acrylic connect app to environment.
- Geometry: consistent shapes/layout reduce cognitive load.
- Typography: hierarchy improves readability + scanning.
- Motion: reactive, context-aware, reinforces way-finding.

## WinUI-specific guidance
- WinUI brings Fluent Design, high-performance rendering, XAML model to C# and C++.
- Prefer WinUI controls/styles for Fluent defaults + accessibility behavior.
