# OS.2020 Development Instructions

## Finding content
os.2020 large. `grep` + glob from broad dirs waste time.
Use #reSearch to find content.
Reading individual files is fine for latest local state because #reSearch does not reflect local changes before PR lands.
Use `grep` + glob ONLY in dirs that match one of these rules:
- The directory is a leaf directory (i.e. it has no subdirectories)
- The directory contains a file named sources
- The directory contains a file named *.vcxproj

DO NOT search from repo root. Command will not finish before user cancels.
If #reSearch MCP not configured or broken, do not try other search paths. Tell user to fix tool.

## OS Branches
reSearch can search many OS branches. Best results: use branch matching local branch. Find it with `git --no-pager log --oneline --merges -5`. Output looks like:

```
3adac4e0b0b6c Git Train: Merge of building/ge_current_directshell_dev5/260216-1630 into official/ge_current_directshell_dev5
37337e5ae87e4 Git Train: Merge of building/ge_current_directshell_dev5/260216-1630 into official/ge_current_directshell_dev5
c9420c30ff6cf Git Train: Merge of building/ge_current_directshell_dev5/260214-1630 into official/ge_current_directshell_dev5
79a532c3a33a3 Git Train: Merge of building/ge_current_directshell_dev5/260214-1630 into official/ge_current_directshell_dev5
4989a17035e84 Git Train: Merge of official/ge_current_directshell into building/ge_current_directshell_dev5/260214-1630
```

Here OS branch = `official/ge_current_directshell_dev5`. Use that branch in #reSearch. `get_codebase_file_matches` expects branch format `ge_current_directshell_dev5 (official)`. `repositoryName` always `Os.2020 (Git)`. `searchScope` looks like `Os.2020 (Git)\\ge_current_directshell_dev5 (official)`.
